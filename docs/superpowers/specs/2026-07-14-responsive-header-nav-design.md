# 반응형 헤더 내비게이션 디자인 사양

## 수정 범위

공용 헤더의 메뉴 버튼을 누르면 피그마 노드 `166:3714`와 동일한 LNB
패널을 표시한다. 데스크톱 내비게이션과 기존 링크 목록은 변경하지 않는다.

## 피그마 사양

- 패널 크기: `320px × 100dvh`
- 패널 배경: `#FFFFFF`
- 패널 바깥 딤드: `rgba(0, 0, 0, 0.8)`
- 패널 내부 여백: `8px`
- 닫기 행과 각 메뉴 행 높이: `52px`
- 행 좌우 여백: `16px`
- 행 둥글기: `16px`
- 메뉴 글자: Pretendard, `16px`, `500`, 줄 높이 `24px`
- 메뉴 글자색과 닫기 아이콘: `#222222`(`그레이_800`)
- 선택 메뉴 배경: `#F6F8FB`(`그레이_50`)
- 닫기 아이콘: `24px × 24px`, `2px` 둥근 선,
  경로 `M16 8L8 16M16 16L8 8`

## 동작

- 메뉴 버튼으로 패널을 연다.
- 처음 열었을 때 모든 메뉴 행은 흰색 기본 상태다.
- 메뉴를 선택하면 선택한 행에만 `#F6F8FB` 배경을 적용하고 다시 열어도
  해당 선택 상태를 유지한다.
- 닫기 버튼, 패널 바깥 영역, `Escape` 키로 패널을 닫는다.
- 메뉴 링크를 선택하면 기존 목적지로 이동하고 패널을 닫는다.
- 패널이 열리면 본문 스크롤을 잠그고 닫힐 때 복원한다.
- `1100px` 이상 데스크톱 너비로 변경되면 열린 패널을 닫는다.
- `1100px`부터 `1439px`까지는 헤더 좌우 여백 `20px`, 메뉴 간격 `16px`을
  사용하고 `1440px` 이상에서는 각각 `40px`, `32px`을 사용한다.
- 메뉴 버튼에 `aria-expanded`와 `aria-controls`를 제공한다.

## 반응형 처리

- 패널은 화면 오른쪽 위에 고정한다.
- 기본 너비는 `320px`이고 화면이 더 좁으면 `100vw`를 사용한다.
- 패널은 화면 높이 전체를 채우며 내용이 넘치면 세로 스크롤할 수 있다.
- 지정된 딤드 외 추가 배경 효과, 그림자, 애니메이션은 사용하지 않는다.

## 확인 기준

- 메뉴 버튼 클릭 후 `320px × 100dvh` 패널과 80% 검은색 딤드가 열린다.
- 최초 상태에는 선택된 메뉴가 없고 클릭한 메뉴만 선택 배경을 가진다.
- 닫기 버튼, 바깥 영역, `Escape`, 링크 선택으로 정상적으로 닫힌다.
- 각 행의 크기, 여백, 색상, 타이포와 아이콘이 피그마 값과 일치한다.
- 린트, 타입 검사와 프로덕션 빌드가 통과한다.
