# 블로그 목록 페이지 설계

## 목표

피그마의 `블로그_P`, `블로그_T`, `블로그_F`, `블로그_M` 프레임을 기준으로 사용자용 블로그 목록 페이지 `/blog`를 구현한다. 이번 범위에는 블로그 상세 페이지와 실제 서버 연결을 포함하지 않는다.

## 피그마 기준

- 기준 노드는 P `33:3195`, T `40:4123`, F `41:4575`, M `41:5097`이다.
- 공용 헤더와 푸터를 유지하고, 블로그 전용 히어로는 공용 `PageHero`의 서브페이지 형태를 사용한다.
- 목록 상단에는 `C · Brain 공식 블로그` 뱃지, 전체 게시글 수, 카테고리 탭을 배치한다.
- 첫 게시글은 큰 대표 카드로 표시하고 나머지는 일반 카드 목록으로 표시한다.
- 데스크톱과 태블릿에서는 일반 카드 2열과 우측 사이드 영역을 사용한다.
- 폴드에서는 일반 카드 2열을 유지하되 실시간 상담 박스를 게시글 사이의 전체 너비 행으로 옮기고 TOP5 영역은 숨긴다.
- 모바일에서는 모든 게시글을 1열로 표시하고 실시간 상담 박스를 게시글 사이에 배치한다.
- 카테고리 탭은 좁은 화면에서 가로 스크롤할 수 있어야 하며 첫 항목과 마지막 항목에 컨테이너 여백을 유지한다.

## 컴포넌트 구조

- `apps/user/app/(site)/blog/page.tsx`: 서버 컴포넌트인 페이지 진입점과 히어로 조합을 담당한다.
- `BlogBoard`: 카테고리 선택 상태와 필터링된 목록을 관리하는 클라이언트 컴포넌트다.
- `BlogFeaturedCard`: 대표 게시글의 이미지 오버레이, 카테고리, 제목, 설명, 순번을 표시한다.
- `BlogCard`: 일반 게시글의 이미지, 카테고리, 제목, 요약, 작성자, 날짜를 표시한다.
- `BlogConsultCard`: 실시간 카카오톡 상담 안내와 버튼을 표시한다.
- `BlogPopularList`: TOP5 게시글 목록을 표시한다.
- 컴포넌트는 게시글 객체를 props로 받고, 화면 문구나 게시글 배열을 내부에 직접 작성하지 않는다.

## 데이터 구조

- 블로그 게시글 타입과 더미 데이터는 각각 `_types`, `_data` 폴더로 분리한다.
- 게시글은 `id`, `category`, `title`, `summary`, `publishedAt`, `author`, `image`, `featured`, `popularRank` 필드를 가진다.
- 카테고리 값은 별도 상수로 관리하고 탭과 데이터가 같은 값을 사용하도록 한다.
- 향후 서버 연결 시 `_data`의 더미 배열만 API 응답으로 교체할 수 있도록 카드 컴포넌트는 데이터 출처를 알지 못하게 한다.
- 이번 범위에서 카드 링크는 향후 `/blog/[id]` 상세 라우트를 연결할 수 있는 URL 구조를 사용하되 상세 페이지는 만들지 않는다.

## 스타일 및 반응형

- 프로젝트의 공통 브레이크포인트 `640px`, `1080px`, `1440px`를 따른다.
- P 구간은 `1360px` 콘텐츠 폭 안에서 본문과 사이드 영역을 구성한다.
- T 구간은 좌우 `20px` 여백을 유지하고 본문 2열과 사이드 영역을 함께 배치한다.
- F 구간은 좌우 `20px`, 게시글 2열, 전체 너비 상담 박스를 사용한다.
- M 구간은 좌우 `20px`, 게시글 1열을 사용한다.
- 이미지 영역은 `next/image`와 고정된 비율을 사용해 로딩 전후 레이아웃 이동을 막는다.
- 제목과 요약은 피그마의 행 수를 넘으면 CSS 말줄임으로 처리한다.
- 탭은 버튼 요소를 사용하고 키보드 포커스 및 `aria-pressed` 상태를 제공한다.

## 공용 영역 변경

- 헤더의 블로그 메뉴는 `/blog`로 연결하고 블로그 경로에서 활성 상태를 표시할 수 있게 한다.
- 랜딩 페이지의 `블로그 전체 보기` 버튼도 `/blog`로 연결한다.
- 기존 랜딩 블로그 섹션의 시각적 스타일이나 데이터는 이번 작업에서 불필요하게 변경하지 않는다.

## 검증

- user 앱 lint, 타입 검사, production build를 실행한다.
- P `1920px`, T `1080px`, F `640px`, M `390px`에서 피그마와 비교한다.
- 작은 데스크톱에서도 카테고리 탭을 마우스 드래그와 트랙패드로 이동할 수 있는지 확인한다.
- 탭 필터링 시 페이지가 최상단으로 이동하지 않는지 확인한다.
- 긴 제목과 요약, 이미지 로딩, 빈 필터 결과에서도 레이아웃이 무너지지 않는지 확인한다.
